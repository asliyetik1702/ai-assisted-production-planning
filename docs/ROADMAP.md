# Project Roadmap

This roadmap outlines the planned development stages of the AI-assisted inventory and production planning decision support system.

## Phase 1 — Data Foundation

- Collect historical inventory and production data
- Review the structure and quality of existing datasets
- Create a standardized SKU master table
- Match product codes with standardized product names
- Identify missing, duplicate and inconsistent records
- Consolidate data from multiple operational sources
- Prepare anonymized sample data for publication

## Phase 2 — Descriptive Analytics

- Define inventory and production KPIs
- Perform ABC inventory classification
- Analyze historical stock movements
- Calculate inventory turnover indicators
- Identify slow-moving and obsolete inventory
- Analyze historical production quantities
- Create initial management reporting views

## Phase 3 — Inventory and Production Planning

- Analyze demand patterns
- Estimate safety stock levels
- Calculate reorder points
- Develop stockout-risk indicators
- Compare inventory requirements with production capacity
- Create production-priority rules
- Analyze planned-versus-actual production performance
- Develop scenario-based planning models

## Phase 4 — AI-Assisted Decision Support

- Detect unusual inventory and production movements
- Identify items requiring managerial attention
- Generate automated management summaries
- Explain significant KPI changes
- Produce structured and explainable recommendations
- Support natural-language questions about dashboard results
- Explore AI-assisted scenario evaluation
- Validate AI-generated insights against analytical results

## Phase 5 — Dashboard Development

- Design the initial dashboard wireframe
- Create inventory overview pages
- Create production planning views
- Add KPI summary cards
- Add product, category and SKU filters
- Visualize inventory risks and production trends
- Add alerts and management-priority panels
- Integrate AI-assisted summaries into the interface

## Phase 6 — Validation and Improvement

- Validate calculations using operational scenarios
- Review results with business users
- Measure forecast and recommendation accuracy
- Improve dashboard usability
- Document assumptions and limitations
- Refine analytical and AI-assisted components
- Prepare the final project demonstration

## Current Focus

The project is currently focused on:

- SKU standardization
- Product-name mapping
- Data consolidation
- Data-quality assessment
- Development of the initial analytical structure
